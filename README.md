# Question & Answer API

## Описание проекта

Это тестовое задание для позиции Python-разработчика. Проект представляет собой REST API сервис для вопросов и ответов, разработанный с использованием FastAPI и PostgreSQL.

## Цели проекта

### Демонстрация навыков
- **Разработка через тестирование (TDD)** - весь функционал реализован через цикл "красный-зеленый-рефакторинг"
- **Чистая архитектура** - разделение на слои: API, схемы, CRUD, модели
- **Современный стек** - FastAPI, SQLAlchemy 2.0, Pydantic v2
- **Профессиональные практики** - типизация, валидация, обработка ошибок

### Соответствие требованиям
- Модульная структура проекта
- Полный CRUD для вопросов и ответов
- Каскадное удаление связанных данных
- Docker-контейнеризация
- Миграции базы данных

## Функциональность

### Вопросы (Questions)
- `GET /questions/` - список всех вопросов
- `POST /questions/` - создать новый вопрос
- `GET /questions/{id}` - получить вопрос с ответами
- `DELETE /questions/{id}` - удалить вопрос (с ответами)

### Ответы (Answers) 
- `POST /questions/{id}/answers/` - добавить ответ к вопросу
- `GET /answers/{id}` - получить конкретный ответ
- `DELETE /answers/{id}` - удалить ответ

## Технический стек

- **FastAPI** - веб-фреймворк
- **SQLAlchemy 2.0** - ORM
- **PostgreSQL** - база данных
- **Pydantic v2** - валидация данных
- **Alembic** - миграции
- **Docker** - контейнеризация
- **pytest** - тестирование

## Архитектурные решения

### Слоевая архитектура
- **API слой** - обработка HTTP запросов
- **Схемы** - валидация входящих/исходящих данных
- **CRUD слой** - бизнес-логика работы с данными
- **Модели** - описание структуры базы данных

### TDD подход
Каждая функция реализована через:
1. Написание падающего теста
2. Минимальную реализацию для прохождения теста
3. Рефакторинг кода

### Современные практики
- Type hints для всего кода
- Современный синтаксис SQLAlchemy 2.0
- Конфигурация через Pydantic settings
- Обработка ошибок на всех уровнях

## Запуск проекта

```bash
# Установка зависимостей
pip install -r requirements.txt

# Запуск тестов
pytest

# Запуск приложения
uvicorn src.main:app --reload
```

## Документация API

После запуска приложения документация доступна по адресу:
- `http://localhost:8000/docs` - Swagger UI
- `http://localhost:8000/redoc` - ReDoc

## Статус разработки

Проект находится в активной разработке согласно требованиям тестового задания. Реализована базовая функциональность вопросов, ведется работа над ответами и каскадными операциями.
